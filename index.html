Introduction to HTML

HTML or Hypertext Markup Language is the backbone of all websites. It provides the structure in which elements of the page are styled and how the user will navigate through the website or app.

Every website follows a basic format which includes a header, a body and a footer. Even if you don't notice it when you regularly use the web it is present and makes the experience enjoyable and easily interactive

Throughout this technical documentartion of HTML we will discuss several concepts including but not limited to: tags, creating lists, and organizing your text code into compartments called divs. Let's get started!

Top Tags

Tags are what is used in HTML to help define what is actually going on with the test. Here I will list several of the most common and most used tags that you can use to start building your very own website.

Code Description
< !DOCTYPE html > Ironically the first one is not a tag but must be included before everything else. It is an instruction to the web browser and tells it what version it is running.
< html > The first tag tells the browser page that it's running HTML
< head > The head tag contains information like scripts, styles and/or even a title for the page
< title > The title tag is required in all HTML and will display the title of the page in the browser window
< body > The body tag will contain all of the elements in the actual html so all of the text, images, lists and so on
< h1 > to < h6 > Now we actually get to writing text for the page. These are heading tags. h1 by default being the biggest in text size all the way down to h6. These are helpful to oranize information.
< p > This tag denotes a paragraph for your text.
< br > Using this tag will add a break to your text since just hitting enter won't actually do anyhing
< hr > This tag will add a horizontal line to your page and help to organize content.

It is IMPORTANT to note that almost all tags (there are a few exceptions) need a closing tag. So for example if you make a paragraph using < p > you need to close the tag after your paragraph like this < /p >

Comment

Adding comments in your HTML is incredibly useful to yourself and other coders in the future who will be reading your code. It allows you to type an instruction or a note in the code without affecting your actual HTML.

To do this you would type < !-- ... -- >

You can type this anywhere in the HTML whether it be before a div tag noting what the div is holding or you could add it after a hex code to show what it is representing. It is up to you how you want to add notes to your code but it is a very good habit to get into.

Attributes

An attribute in HTML can do two things:

  1. Modifies default functionality of the element type
  2. Provides functionality to certain element types unable to function correctly without them

Generally the syntax for attributes is as follows < element attribute="value" > Content in the tag < /element >

Some examples of attributes include:

  • < src > which stands for source. You'll use this and the link for an image you want to add.
  • < href > will make text into a clickable link
  • < id > gives an element a unique identification which becomes useful when you start using CSS
  • < class > is another common attribute used in assisting your CSS code
  • < width > and < height > are two attributes that do exactly what you think they do. Very useful in sizing images or other content.
Adding Images to Your Website

Images are a great way to make your website really stand out from the rest. From getting the pictures off the internet, sizing them and even turning them into a link you can make your page stand out from the rest.

We talked earlier about tags and how they need to be closed. Well this tag is one of the exceptions. The img tag does not need to be closed. It is referred to as self-closing.

The common syntax for adding an image

< img src= " " >

Now there are many things you can do with said image once you've got it on your page. You can control the height and width using the attributes we went over earlier in the last section

height = "100px" width = "100px"

This will make your image 100 pixels tall by 100 pixels wide.

Another cool thing you can do is turn your image into a link like this

< a href = "Linktopicture" >

And once you have your link you put the a tag around the image tag so once you have all of the subsequent images in place it will look like this

< a href = "link" >
< img src="picture-link" width = "100px" height = "10px" >
< / a >

These are just some of the basics when it comes to images. It is up to YOU what images you want to use and how you want to style them.

NOTE: If for some reason your picture won't load it is always recommended you include an alt attribute at the end of the picture with a short description of what it is. So for example if I linked a picture of a child holdin a ballon and it didn't load my alt attribute would take it's place in plain text.

Format Text

You may find it useful to format your text when you want to make something bold or italic. Thankfully, there are some really simple and helpful tags to help make certain words stand out from the rest.

  • < strong > makes your text look important
  • < i > will give the text an italic style
  • < mark > will highlight text for you

Proper syntax for formatting text is nesting the word inside the tag. It will look something like this

< b > This text will be bold < / b >

And that works for any of the format tags. You can even nest a formatted text within another tag.

Creating Lists

If you couldn't tell from this Technical Document page I love lists and you will too once you know how to create them in HTML. There are two different kinds of lists

  1. Ordered Lists
  2. Unordered Lists

Ordered lists will have a number next to them and unordered will have a dot. You can always go to CSS and format the numbers and dots how you want but for now we will just stick to making the default lists.

To make the list you use this block of code

< ol >
< li > List Item #1 < /li >
< li > List Item #2 < /li >
< li > List Item #3 < /li >
< /ol >

So as you can see there's a couple different things we have going on here. First, we open up our ordered list tag. Second, we have our list items in the opening and closing li tags. Finally, we have to close the ordered list tag. If you wanted an unordered list you would just use

< ul > < /ul >

But still use the same li tags in the list to actually create the list.

Block Elements

It is very important to organize in your HTML because once you introduce CSS and JavaScript you will start to run into conflicting id's and classes. Adding block elements or the div tag gives your page:

  • more structure and organization
  • a container to style CSS or perform certain tasks with JavaScript

Div tags work like this

< div > < / div >

And you can put content inside the div tags like a heading or an image for example.

Nesting

Nesting elements becomes very important when you want to put similar information together all in one container or div. Doing this allows you to keep your HTML looking tight and organized.

To nest items it looks like this.

< div >
< h1 > Hello World < /h1 >
< p > This is a paragraph < /p >
< /div >

So basically you can put as many elements as you want into a single div and then they will be considered nested!

Building a Form

By building a form with just HTML you open up the option for the user to input any information (usernames, passwords, email) and have that information sent to your database. Let me show you how in a couple of quick steps

First you want to make the form

< form action="/url-where-you-want-to-submit-form-data" > < /form >

Then you want to add a button so your user can submit their information to you

< button type="submit">this button submits the form < /button >

And when you put it all together with the text field nested in the form element you will have something that looks like this

< form action="/submit-name" >
< input type="text" placeholder="Your Name Here" >
< button type="submit" > Submit < /button >
< /form >

And now you have yourself an element where a user can enter their name or anything. The placeholder will give the form a pre-set text so it gives your user some idea of where to put the information and in what way. You're totally in control!
